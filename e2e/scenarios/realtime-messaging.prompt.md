## RTM-001: 用户消息实时推送到管理员页面

**优先级：** P0

**前置条件：**
- 用户已登录（testuser）
- 管理员已登录（admin）
- OpenCode 服务正常运行
- 存在一个进行中的会话（可由 TC-CHAT-001 创建）

**测试步骤：**

1. 管理员页面：
   - 使用 `chrome-devtools_navigate_page` 打开管理后台 http://localhost:3000/admin
   - 使用 `chrome-devtools_take_snapshot` 获取页面快照
   - 点击进入某个进行中的会话详情
   - 记录当前消息数量

2. 用户页面：
   - 在另一个浏览器标签页打开 http://localhost:3000
   - 导航到同一个会话
   - 发送消息：`这是实时测试消息`

3. 管理员页面验证：
   - 使用 `chrome-devtools_wait_for` 等待新消息出现
   - 使用 `chrome-devtools_take_snapshot` 验证消息已显示
   - 确认无需手动刷新页面

**预期结果：**
- 管理员页面在 2 秒内自动显示用户的新消息
- 消息内容为"这是实时测试消息"
- 消息发送者为用户

**验证点：**
- 管理员页面消息数量增加 1
- 新消息显示在消息列表中
- 消息时间戳正确
- 无需刷新页面

- 消息发送者标记为"管理员"

**验证点：**
- 用户页面消息数量增加 1
- 新消息显示在左侧（管理员消息位置）
- 消息发送者显示为"管理员"

## RTM-004: 会话状态变更实时同步

- 管理员已登录（admin）

1. 管理员页面：

2. 用户页面：

   - 点击"标记为需要人工处理"按钮

   - 使用 `chrome-devtools_take_snapshot` 验证状态已更新

**预期结果：**
- 管理员会话列表中该会话状态自动变为"待人工"
- 状态标签颜色变为橙色

**验证点：**
- 会话状态标签显示"待人工"
- 状态颜色为橙色
- 无需刷新页面

## RTM-005: SSE 断线重连

**优先级：** P2

**前置条件：**
- 用户已登录（testuser）
- 管理员已登录（admin）
- 存在一个进行中的会话

**测试步骤：**

1. 管理员页面：
   - 打开会话详情页
   - 使用 `chrome-devtools_evaluate_script` 检查 SSE 连接状态
   - 记录初始连接状态

2. 模拟网络中断：
   - 使用 `chrome-devtools_emulate` 设置网络为离线模式
   - 等待 5 秒

3. 恢复网络：
   - 使用 `chrome-devtools_emulate` 恢复网络
   - 等待 3 秒

4. 用户页面：
   - 发送新消息：`重连测试消息`

5. 管理员页面验证：
   - 使用 `chrome-devtools_wait_for` 等待新消息出现
   - 使用 `chrome-devtools_take_snapshot` 验证消息已显示

**预期结果：**
- SSE 连接自动重连
- 重连后能正常接收新消息
- 消息内容正确显示

**验证点：**
- SSE 连接状态恢复为"已连接"
- 新消息在 3 秒内显示
- 无需手动刷新页面
